## TASK C — добавление скраперов страниц туров артистов (НОВАЯ ОСНОВНАЯ ЗАДАЧА — имеет приоритет над Task A)

**Зачем это нужно:** скраперы площадок и обход Ticketmaster покрывают только те рынки/площадки,
о которых мы уже знаем (в основном Европа/Северная Америка). Пользователь заметил реальный
пробел вживую: The Weeknd появлялся для Сингапура, но не для Японии — не из-за бага, а потому
что ни один источник данных не имеет реального присутствия в Японии. Настоящее решение для
«дай мне каждый концерт МОИХ артистов, где бы они ни играли» — это скрапер, ключом для которого
служит АРТИСТ, а не площадка/страна — их собственная официальная страница тура или (если их
сайт встраивает его) публичный виджет Bandsintown/Songkick на их сайте. Это обычная загрузка
публичной веб-страницы, а не закрытый API для разработчиков — та же легитимность, что и при
скрапинге любой другой страницы в этом репозитории.

**Список:** `data/artist_scrape_targets.txt`, по одному имени артиста/группы на строку, ~1400
записей (смесь английских и русских имён — пройди через все, по порядку). Некоторые записи —
шум (жанровые теги, случайные слова вроде «билборды», не настоящие артисты) — если имя явно не
идентифицирует реальный гастролирующий коллектив, ПРОПУСТИ его и отметь это, не гадай.

**Расположение конфигов отличается от Task A:** конфиги артистов идут в `scrapers/artists/`
(не в директорию верхнего уровня `scrapers/`) — это намеренно, они выполняются по собственному
еженедельному расписанию (`artist-scrape.yml`), а не по ежедневному, так что нет давления по
времени, как у площадок в Task A. `npm run test-config -- <id>` уже проверяет оба расположения,
дополнительный шаг не нужен.

**Возобновляемость:** каждый артист становится ровно одним файлом,
`scrapers/artists/<artist-slug>-tour.json`. Перед тем как работать над артистом, проверь, не
существует ли уже этот файл — если да, пропусти его (уже сделано, независимо от того, тобой или
предыдущей сессией). Это делает весь список безопасно возобновляемым между сессиями/
перезапусками без необходимости отдельного файла для отслеживания прогресса.

**На каждого артиста:**
1. Возьми следующего артиста из списка, для которого `scrapers/artists/<slug>-tour.json` ещё
   не существует.
2. Найди его настоящую официальную страницу тура/дат: сначала попробуй `<official-site>/tour`,
   `/shows`, `/live`, `/events`. Если на его собственном сайте нет списка тура, но встроен
   виджет Bandsintown или Songkick — это тоже валидный источник.
3. Загрузи её, выбери `type` в ТОМ ЖЕ порядке предпочтения, что и в Task A: `jsonld` >
   `next_data` > `json_api` > `static_selectors` (последний вариант; если странице нужен
   рендеринг JS, чтобы показать события, используй `playwright_render` вместо того чтобы
   сдаваться).
4. Запиши `scrapers/artists/<artist-slug>-tour.json`. Та же схема/шаблон, что и у конфигов
   площадок из Task A (см. ниже), ЗА ИСКЛЮЧЕНИЕМ того, что нет фиксированной площадки/города —
   если страница тура перечисляет несколько городов/площадок в рамках тура артиста, это
   нормально, движок уже умеет обрабатывать несколько событий на скрапер; выставляй
   `venueNameFallback`/`cityNameFallback` только если страница не предоставляет надёжно свою
   собственную площадку/город на событие (не указывай их в селекторах и дай реальным данным
   страницы пройти сквозь, если они там есть).
5. Тест: `npm run test-config -- <artist-slug>-tour` → должно вывести `OK — N events`, где
   N≥1, и реальную дату. Если тест не проходит или у артиста нет доступного публичного списка
   тура нигде (нет официального сайта, нет виджета, ничего) — ПРОПУСТИ и отметь это, не
   фабрикуй конфиг, который на самом деле не работает. (Сфабрикованный скрапер, который всегда
   выдаёт 0 событий или ENOTFOUND, уже был пойман и удалён один раз в этой сессии — не повторяй
   это.)
6. `npm test && npx tsc --noEmit` зелёные → коммить `feat(scrapers): add <artist> tour page`.

Проходи список по порядку. Это высокообъёмная работа, как и Task A — продолжай.

## TASK A — добавление скраперов площадок (делай это, если Task C полностью проработан)

Полный рецепт: `docs/ADD-VENUE-SCRAPERS.md`. Прочитай его один раз. Затем повторяй этот цикл:

scripts package.json. Коммить `chore: add test-config helper`.

**На каждую площадку:**
1. Найди реальную концертную площадку с публичным URL расписания/афиши. Города для охвата:
   Amsterdam, Berlin, Belgrade, Tbilisi, London (затем любые другие).
2. Загрузи её. Выбери `type` в этом порядке:
   - `"jsonld"` — на странице есть `<script type="application/ld+json">` с
     `"@type":"MusicEvent"`/`"Event"`. Лучший вариант. CSS не нужен.
   - `"next_data"` — на странице есть `id="__NEXT_DATA__"`; помести JSON-пути в selectors.
   - `"static_selectors"` — выбери СТАБИЛЬНЫЕ CSS-селекторы. **Избегай хэшированных классов**
     вроде `css-8tb23n`.
3. Запиши `scrapers/<venue>-<city>.json`, используя шаблон ниже.
4. Тест: `npm run test-config -- <venue>-<city>` → должно вывести `OK — N events`, где N≥1
   и разумные артист + дату в примерах. Если не проходит, исправь селекторы/type или пропусти
   площадку (отметь это).
5. `npm test && npx tsc --noEmit` зелёные → коммить `feat(scrapers): add <venue> (<city>)`.

**Шаблон:**
```json
{
  "id": "paradiso-amsterdam",
  "domain": "paradiso.nl",
  "url": "https://www.paradiso.nl/en/agenda",
  "type": "jsonld",
  "selectors": {
    "eventBlock": "",
    "date": "",
    "venueNameFallback": "Paradiso",
    "cityNameFallback": "Amsterdam",
    "countryNameFallback": "NL"
  }
}
```
Правила для конфига: `id` должен соответствовать `^[a-z0-9][a-z0-9-]*$` (только строчные
буквы, цифры, дефисы — БЕЗ слэшей, точек или `..`). `countryNameFallback` — ровно 2 буквы
(код ISO). Для `static_selectors` обязательны `eventBlock`/`date`, а `artist` обычно тоже
нужен.

Продолжай — это твой основной результат работы. Стремись охватить много площадок.
